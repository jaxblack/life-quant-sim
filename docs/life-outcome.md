# LifeOutcome — 模拟终点的多维评估

> **核心理念**：人生模拟器的终点 (LifeOutcome) 不是单一的"赚了多少钱"，
> 而是一个 **多维向量**。金钱只是其中一个维度，且未必权重最大。
> 我们用 5 个维度刻画一个 agent 走完一生后的"成色"，再用可配置权重
> 折算成 0–100 的综合评分供横向比较。

---

## 1. 总金钱价值 `total_money_value`

agent 全生命周期累计的可支配财富（包含现金、净投资资产、未实现收益的合理折现）。
**对数化处理**：财富分布天然长尾，原始数值会让百万富翁与亿万富翁差异
压倒一切其他维度，因此评分时取 log 曲线、超过 1e6 后饱和。

- **公式草案**：`money_score = clamp( log10(v + 1) / 6 * 100, 0, 100 )`
- **取值范围**：`v ∈ [0, +∞)` 美元当量；归一化后 `[0, 100]`
- **说明**：v = 1e6 → ~100 分；v = 1e3 → ~50 分；v ≤ 0 → 0 分

---

## 2. 总社会价值 `total_social_value`

agent 一生中**对他人产生的正向影响总量**，近似为 `被影响人数 × 平均影响深度`。
"影响"包括但不限于：直接帮助、知识传递、雇佣创造、公共贡献。
本维度抵消"自私致富"路径，让"普通教师 / 社工 / 开源作者"也能拿高分。

- **公式草案**：`social = Σᵢ (depthᵢ × decayᵢ)`，其中 `decayᵢ` 是时间衰减因子
- **取值范围**：归一化到 `[0, 100]`，与 `social_capital` 量纲一致
- **说明**：当前实现里这是已经聚合好的 `social_capital`，无需再变换

---

## 3. 自我意义感 `self_meaning`

agent 主观的 "活得是否有意义" —— 由两部分合成：
1. **主观幸福**：日常情绪正负向加权平均（参考 SWB / Ed Diener）
2. **Maslow 高层满足度**：尊重、自我实现、超越性需求的累计达成

这个维度让"内心富足但物质平平"的路径同样有价值。

- **公式草案**：`meaning = α · SWB + (1-α) · MaslowHigh`，默认 `α = 0.4`
- **取值范围**：归一化到 `[0, 100]`
- **说明**：低于 0 截断为 0；满足"心流时间占比 > 30%"额外加成

---

## 4. 健康寿命 `healthy_lifespan`

agent **无重大慢病** 的存活年数 (HALE — Healthy Life Expectancy)。
不是 raw lifespan：90 岁但 60 岁起卧床的 agent 健康寿命只有约 60。
这个维度直接抑制"以健康换金钱"的极端策略。

- **公式草案**：`lifespan_score = clamp(years / 101 × 100, 0, 100)`
- **取值范围**：`years ∈ [0, 101]`；归一化后 `[0, 100]`
- **说明**：101 是人类历史平均上限的设定值；超过则饱和

---

## 5. 关系网密度 `network_density`

agent 在生命终点拥有的**社会关系网络结构特征**，由三部分加权：
- **强弱关系总数**：紧密 friends + 弱联系 acquaintances
- **Dunbar 数贴合度**：稳定关系数是否落在 ~150 的认知上限合理区间
- **跨阶层桥连接数**：连接不同社会层的"结构洞"数量（Burt 1992）

孤独是早死最强预测因子之一，因此本维度独立于 `social_value`：
你可以"影响很多陌生人"但仍"无亲密朋友"。

- **公式草案**：`net_density = β·log(strong+1) + γ·dunbar_fit + δ·bridges`
  默认 `β=0.4, γ=0.3, δ=0.3`
- **取值范围**：归一化到 `[0, 100]`
- **说明**：当前为**评估维度**（已记录于 outcome），但综合评分公式 v1
  尚未纳入 —— 见下方"综合评分"备注

## 综合评分 (0–100)

将上述维度折算为单一可比分数：

```
score = w_money · money_s
      + w_social · social_s
      + w_meaning · meaning_s
      + w_lifespan · lifespan_s
```

**默认权重**（与 `engine/scoring.py:DEFAULT_WEIGHTS` 对齐）：

| 维度       | 权重 |
|------------|------|
| money      | 0.25 |
| social     | 0.25 |
| meaning    | 0.30 |
| lifespan   | 0.20 |
| **合计**   | 1.00 |

**约束**：weights 字典必须包含全部 4 个 key，且和为 `1.0 ± 1e-6`，否则 `scoring.score_outcome` 抛 `ValueError`。

**关于 `network_density`**：本文档 5 维评估覆盖完整概念模型，
但综合评分公式 v1 仅纳入前 4 个维度（与 `scoring.py` 实现一致）。
`network_density` 已作为 outcome 字段记录，留待 v2 纳入加权
（拟定权重 `0.10`，其它维度等比缩放）。
